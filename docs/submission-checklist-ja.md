## E. 実機スモーク（提出直前）

新規インストール推奨（初回ヒント確認用）。

### 起動・設定
- [ ] 初回のみ「明るさ」「言語」の説明シートが出る（既存端末では出ない）
- [ ] 右上でライト/ダーク、EN/JP が切り替わる
- [ ] ℹ️ → クレジット・出典が開く
- [ ] クレジット内のプライバシー / 利用規約リンクがブラウザで開く
- [ ] お問い合わせ（mailto）が起動する

### 地図・島
- [ ] 日本地図で各地域ピンが選べる
- [ ] ホーム下部に諸島のカバー写真カード（横スクロール）
- [ ] 最近見た島が1件でも横長に伸びない（固定幅カード）
- [ ] 島詳細ヘッダー・背景・クレジットが表示される

### 天気
- [ ] 天気が取得できる（またはキャッシュ／失敗文言）
- [ ] 波の高さ（取得できる島）
- [ ] Open-Meteo 出典リンク
- [ ] 「○分前に取得…」などキャッシュ文言（2回目以降）

### ダイヤ
- [ ] 船便見出しにフェリーアイコン＋青系
- [ ] **八重山**: GTFS の船便時刻がアプリ内に表示される（または取得失敗時の文言）
- [ ] **八重山以外**（例: 神津島・佐渡）: 船便・航空便は **公式リンクのみ**（アプリ内に時刻表なし）
- [ ] 八重山のみ、ダイヤ画面上部に欠航・遅延バナー（任意確認）
- [ ] 各社の「運行状況」「ダイヤ」リンクが外部ブラウザで開く

### 店舗・写真・位置
- [ ] 店舗タブで Apple マップ結果が出る（または圏外失敗）
- [ ] 位置情報許可ダイアログの文言が用途どおり
- [ ] 写真メモ: カメラ／ライブラリ許可文言が用途どおり
- [ ] 写真が端末内に保存され、アプリ削除で消える想定である

### オフライン（可能なら）
- [ ] 機内モードで、以前開いた島はキャッシュ表示
- [ ] 未取得データは短時間で失敗表示になる

## よくある落とし穴

1. 規約 URL を古い `opaquu.github.io` のまま Connect に入れる → 404 でリジェクトリスク  
2. 説明文に **代表ダイヤをアプリ内表示** と書く → 実装（リンクのみ）と不一致  
3. 説明文に WBGT を残す → 機能と不一致  
4. `docs` を更新したのに GitHub Pages へプッシュしていない → アプリ内リンクが古い内容のまま  
5. 英語モードのスクショを使う場合、一部日本語が残ることを把握する（全面バイリンガルではない）  
6. ローカルだけの未コミット変更で Archive し、リモートと食い違う
